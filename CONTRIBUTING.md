# Contributing

Before submitting changes to this repository, follow the [Repository Change Checklist](docs/repo-change-checklist.md).

At minimum, every change should:

- Update related documentation.
- Update the GitHub Pages landing page when relevant.
- Validate affected tools, scripts, and generated artifacts.
- Update sparse clone behavior when new files should be included in toolkit or platform profiles.
- Avoid committing secrets, tenant-specific IDs, customer data, or private PBIP artifacts.
- Be committed with a clear message so the change can be rolled back if needed.

For tool changes, also update:

- `tools/index.html`
- `tools/README.md`
- `README.md`
- `docs/governance/power-bi-governance-tools.md`
- `TODO.md`
